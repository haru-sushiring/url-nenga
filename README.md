- 中央寄せにし、最低限の使いやすさを目指しました。
- 縦：445px、横：300pxであれば、年賀状の表面と丁度よい大きさで表示できること。

- 画像をアップロードする際に、1つのボタンを押してから次のボタンが表示されるようにし、迷わず利用できるデザイン設計にしたこと。

## 自己評価
60点
- デザインにもう少しこだわっても良かったが、まずは完成させることを目標にしていたので、完成できた点では良いと思う。
- Firebaseに対して、完全に無知の状態から4つの機能（Authentication、Storage、Realtime Database、Hosting）を利用して完成できたので頑張った方だと思う。
- 後で見返した時に、何の処理をやっているかすぐに分からないので、適切なコメントを付けるべきだった。
- 画像のアップロードに5秒程度かかってしまうので、処理を速めるか、「アップロード中」と表示させるようにしたい。
- ログイン後、過去にアップロードした画像の一覧を見れたり、削除したりできる仕様を追加したい。
- 年賀状の表面の画像に黒い枠線を入れると、ハガキっぽく見やすくなるかもしれない。
- 表面の年賀状に宛名を書けると面白くなりそう。（とある企業の人事の方のアイディア）

# URLねんが
  
## 概要
画像をアップロードすると、画像を年賀状化できて共有できるWebサービスです。  
無料で遊べます。

## URL
[https://nenga.sushiringblog.com](https://nenga.sushiringblog.com)

## 使用技術
- HTML
- CSS
- JavaScript
- PHP
- Bootstrap
- Firebase

## 作った理由
母が、画像加工アプリを使って年賀状用の画像を作り、画像だけLINEで送っていました。  
年賀状を作る上で、印刷する手間と年賀はがきを買いに行く手間がかかるという課題を発見しました。  
そこで「加工した画像を年賀状の表面と組み合わせて送れれば面白いのではないか」と考え、URLねんがを作成しました。

## 工夫点
1. 共有URLを長めのハッシュ値にすることで、どのURLにどんな画像がアップロードされているかを分かりにくくし、ユーザーのプライバシーを守る仕様にしました。
2. 完成した画像をクリックすると、年賀状がクリックするごとに向きが変わるようにし、遊び要素も追加したこと。
3. アップロード画面はログインしたユーザーだけが見れるようにしたかったため、ログインしていないユーザーは403エラーページに飛ばす仕様にしました。

## 使い方
1. グーグルアカウントで登録する
2. 画像をアップロードする
3. 画像が表示されるまで待つ（5秒ほどお待ち下さい...）
4. 「公開する」を押す
5. 「確認する」を押す
6. アップロードした画像をクリックすると、年賀状の表面の画像が出てきます
7. 「URLをコピーする」を押すと、色んな人と共有できます
